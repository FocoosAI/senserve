# SPEC-000-keep ‚Äî KEEP conventions in this repository

This spec is a self-anchor: it lives inside the knowledge layer so that future maintainers (human or agent) can answer *"how does this repo organize its knowledge?"* by reading the knowledge layer itself, not only the skill under `.agents/skills/keep/`.

If the KEEP skill is ever uninstalled, removed, or replaced, this spec remains as the authoritative description of the convention the repo committed to.

## Goal

Capture the conventions a contributor needs to know to add, update, or query knowledge in this repo without re-deriving them from the skill source.

## Requirements

### Directory layout

```
/knowledge
‚îú‚îÄ‚îÄ docs/
‚îÇ   ‚îú‚îÄ‚îÄ specs/<domain>/<slug>.md     ‚Üê behavioral specifications
‚îÇ   ‚îî‚îÄ‚îÄ decisions/ADR-NNNN-<slug>.md ‚Üê architecture decision records
‚îú‚îÄ‚îÄ ideas/<YYYY-MM-DD>-<slug>.md     ‚Üê parked thoughts, not yet promoted
‚îî‚îÄ‚îÄ INDEX.md                          ‚Üê auto-generated, do not hand-edit
```

Runbooks and architecture docs are *specs with reserved tags* (`runbook` / `architecture`), not separate folders. Active work in progress lives in the ticket tracker, not here.

### Mandatory YAML frontmatter

Every file under `docs/` and `ideas/` starts with:

```yaml
---
id: <stable, immutable id>
title: <short human title>
description: <search snippet ‚Äî what /keep-ask matches against>
status: draft | accepted | deprecated | superseded
type: spec | adr | idea
domain: <area>
tags: [...]
related: [<adr:|spec:|test:|code:|idea:>...]
---
```

The `description` field is load-bearing ‚Äî it is the string `/keep-ask` matches against to decide whether to open the file. Write it as something a future reader would actually type to search for the file.

### Anchors (enforceable claims)

A spec MAY declare anchors that bind a claim to a specific symbol in code. Drift detection (`.agents/skills/keep/scripts/check_drift.py`) verifies these deterministically ‚Äî no LLM. Anchors are opt-in per spec; specs without anchors still exist, they just sit outside the drift gate.

Anchor kinds: `const`, `function`, `test`, `manual`. See `.agents/skills/keep/references/file_formats.md` for the full schema. Anchor the 3‚Äì7 values that *must* hold, not every value that *currently* holds.

### ADR supersession

When ADR-MMMM replaces ADR-NNNN: new ADR has `status: accepted` and `related: [adr:ADR-NNNN]` with a `supersedes` note. Old ADR gets `status: superseded` and a `## Superseded by` section at the end. Body of the old ADR is never edited ‚Äî it is historical record. Partial changes use `## Refined by` on the older ADR (status stays `accepted`). INDEX.md regenerates via `build_index.py`.

### Convention vs. truth

`/knowledge/` is **the authoritative source of truth for the intent of this system**. Code is the authoritative source of truth for the current implementation. When they conflict, `/keep-check-drift` surfaces the conflict; the team decides which one updates.

## The seven commands

- `/keep` ‚Äî dashboard. If `/knowledge` exists, prints stats. If missing, suggests `/keep-init`.
- `/keep-init` ‚Äî bootstrap. Scaffolds `/knowledge`, writes this spec, appends KEEP snippet to AGENTS.md.
- `/keep-ask <question>` ‚Äî read with citations. Consult `/knowledge` before answering behavior/design questions.
- `/keep-compile [source] [--dry]` ‚Äî classify a diff and update specs/ADRs/INDEX. Proposes anchors from concrete values in the diff.
- `/keep-check-drift` ‚Äî deterministic anchor verifier. Exit 1 blocks merge.
- `/keep-idea <thought>` ‚Äî capture a parked thought as `type: idea`, `status: draft`.
- `/keep-govern` ‚Äî periodic hygiene. Surfaces contradicting ADRs, stale ideas, oversize specs.

## Edge cases

- **Skill uninstalled.** This spec remains. Manual maintenance is fine ‚Äî frontmatter, INDEX regeneration, and supersession protocol are human-doable.
- **`/knowledge` removed.** Re-adopt by running `/keep-init` again.

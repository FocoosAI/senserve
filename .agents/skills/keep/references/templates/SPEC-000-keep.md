# SPEC-000-keep — KEEP conventions in this repository

This spec is a self-anchor: it lives inside the knowledge layer so that future maintainers (human or agent) can answer *"how does this repo organize its knowledge?"* by reading the knowledge layer itself, not by reading the skill that authored it.

If the KEEP skill is ever uninstalled, removed, or replaced, this spec remains as the authoritative description of the convention the repo committed to.

## Goal

Capture the conventions a contributor needs to know to add, update, or query knowledge in this repo without re-deriving them from the skill source.

## Requirements

### Directory layout

```
/knowledge
├── docs/
│   ├── specs/<domain>/<slug>.md     ← behavioral specifications
│   └── decisions/ADR-NNNN-<slug>.md ← architecture decision records
├── ideas/<YYYY-MM-DD>-<slug>.md     ← parked thoughts, not yet promoted
└── INDEX.md                          ← auto-generated, do not hand-edit
```

Runbooks and architecture docs are *specs with reserved tags* (`runbook` / `architecture`), not separate folders. Active work in progress lives in the ticket tracker, not here.

### Mandatory YAML frontmatter

Every file under `docs/` and `ideas/` starts with:

```yaml
---
id: <stable, immutable id>
title: <short human title>
description: <search snippet — what /keep-ask matches against>
status: draft | accepted | deprecated | superseded
type: spec | adr | idea
domain: <area>
tags: [...]
related: [<adr:|spec:|test:|code:|idea:>...]
---
```

The `description` field is load-bearing — it is the string `/keep-ask` matches against to decide whether to open the file. Write it as something a future reader would actually type to search for the file.

### Anchors (enforceable claims)

A spec MAY declare anchors that bind a claim to a specific symbol in code. Drift detection (`scripts/check_drift.py`) verifies these deterministically — no LLM. Anchors are opt-in per spec; specs without anchors still exist, they just sit outside the drift gate.

Anchor kinds: `const`, `function`, `test`, `manual`. See `<skill-path>/references/file_formats.md` for the full schema. The rule of thumb: anchor the 3-7 values that *must* hold, not every value that *currently* holds.

### ADR supersession

When ADR-MMMM replaces ADR-NNNN: new ADR has `status: accepted` and `related: [adr:ADR-NNNN]` with a `supersedes` note. Old ADR gets `status: superseded` and a `## Superseded by` section at the end. Body of the old ADR is never edited — it is historical record. INDEX.md regenerates the supersession chain automatically.

### Convention vs. truth

`/knowledge/` is **the authoritative source of truth for the intent of this system**. Code is the authoritative source of truth for the current implementation. They answer different questions and both can be right.

When they conflict, drift detection surfaces the conflict; the team decides which one updates.

## The seven commands

The KEEP skill exposes seven slash commands. Their full contracts live in the skill's `commands/` directory; this is a one-liner index:

- `/keep` — dashboard. If `/knowledge` exists, prints stats. If missing, suggests `/keep-init`.
- `/keep-init` — bootstrap. Scaffolds `/knowledge`, writes this spec, appends KEEP snippet to AGENTS.md.
- `/keep-ask <question>` — read with citations. The agent consults `/knowledge` before answering.
- `/keep-compile [source] [--dry]` — classify a diff and update specs/ADRs/INDEX in one shot. Proposes anchors from concrete values in the diff.
- `/keep-check-drift` — deterministic anchor verifier. Exit 1 blocks merge. CI-friendly.
- `/keep-idea <thought>` — capture a parked thought as `type: idea`, `status: draft`.
- `/keep-govern` — periodic hygiene (run weekly). Surfaces contradicting ADRs, stale ideas, oversize specs.

## Edge cases

- **Skill uninstalled.** This spec remains. The conventions still apply to anyone (human or agent) reading the repo; they just don't have the slash commands. Manual maintenance is fine — frontmatter, INDEX regeneration, and supersession protocol are all human-doable.
- **`/knowledge` removed.** The repo opted out of KEEP. Re-adopt by running `/keep-init` again.
- **Conflicting AI entry files.** If both `AGENTS.md` and `CLAUDE.md` exist, the KEEP snippet is appended to `AGENTS.md` (preferred) — agents should read both.
